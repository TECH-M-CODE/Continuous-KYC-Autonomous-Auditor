## 1. Problem Statement
Traditional Know Your Customer (KYC) and Anti-Money Laundering (AML) audits are highly manual, point-in-time processes. Compliance teams are overwhelmed by false-positive alerts generated by legacy rules engines, resulting in alert fatigue and delayed detection of real risks (e.g., rapid sanctions evasion or sudden adverse media exposure).

## 2. Solution Overview
CXKYC is an autonomous, event-driven auditing system that continuously monitors entities against live data feeds (news, sanctions, financial events). Using a hybrid approach of deterministic fuzzy-matching pre-filters and a specialized Multi-Agent LLM network, the system triages risks in real-time, drafts Suspicious Activity Reports (SARs) backed by verifiable evidence, and ensures a human is always in the loop for final decisions.

## 3. Key Features
- **Continuous Monitoring:** Ingests live data from news feeds and sanctions lists.
- **Multi-Agent Orchestration:** Specialized AI agents (Investigator, Resolver, Reporter) evaluate risks autonomously.
- **Explainable AI:** LLM decisions are grounded in actual evidence and regulatory rules using Retrieval-Augmented Generation (RAG).
- **Append-Only Audit Trail:** Every action (AI or human) is permanently recorded in a tamper-evident, hash-chained log.
- **Human-in-the-Loop:** No SAR is filed without explicit compliance officer sign-off.

## 4. Tech Stack
- **Frontend:** Vanilla JS, ReactFlow (Decision Graphs), Server-Sent Events (SSE)
- **Backend:** FastAPI (Python), APScheduler
- **AI/LLM:** Google Gemini via LLM Gateway, LangGraph (Agent orchestration)
- **Knowledge/Vector:** ChromaDB (RAG collections)
- **Data/Storage:** SQLite (WAL mode), SQLAlchemy ORM

## 5. Architecture Diagram
*(See SYSTEM_DESIGN.md for complete diagrams)*
The system runs in three concurrent loops:
1. **Ingestion Loop:** Feed Adapters fetch and normalize events.
2. **Processing Loop:** LangGraph agents evaluate events and update entity risk scores.
3. **Human Review Loop:** Compliance officers review alerts, verify evidence, and sign off on SARs.

## 6. Quick Setup
### Prerequisites
- Python 3.11+
- Node.js (for frontend dependencies)
- Gemini API Key

### Steps
1. **Clone the repository.**
2. **Setup Backend:**
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # On Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   cp .env.example .env  # Add your GEMINI_API_KEY
   python data/seed/seed_entities.py  # Seed the initial database
   uvicorn app.main:app --reload
   ```
3. **Setup Frontend:**
   ```bash
   cd frontend
   npm install
   npm run dev
   ```
